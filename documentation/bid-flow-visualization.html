Command

5 placeBid orchestration src/server/auction-commands.ts

The heart of the write path. Before opening a transaction it:

  • Validates the payload with bidInputSchema.parse(input) — see step 6.
  • Calls advanceAuctionLifecycle() (auction-lifecycle.ts) to auto-close any expired auctions first.
  • Opens an OpenTelemetry placeBid span and starts a duration timer.

6 Shared Zod schema src/domain/validation.ts

bidInputSchema enforces UUIDs, a positive integer amountCents, and a nullable expectedHighestBidCents. The same schema is shared with the client form — one source of truth.

DB · lock

7 Row lock src/db/client.ts

SELECT id FROM auctions WHERE id = $1 FOR UPDATE serialises concurrent bids on the same auction so two buyers can’t both win.

DB · read

8 Load current state

Inside the locked tx it reads the auction (with fish item + seller), the committed current highest bid (ORDER BY amountCents DESC), and the bidder record.

Domain rules

9 evaluateBid src/domain/bid-rules.ts

Pure decision function (uses bid-builder.ts for the minimum-increment floor). Returns a typed rejection or acceptance, checking in order:

  • INVALID_AMOUNT — not a positive integer of cents
  • AUCTION_NOT_ACTIVE — status/clock window
  • SELLER_OWN_AUCTION — seller can’t bid on own fish
  • STALE_BID — expected ≠ committed highest
  • INSUFFICIENT_INCREMENT — below the next valid floor
DB · write

10 Persist the outcome src/db/schema.ts

If rejected → INSERT INTO rejected_bids (audit trail) and return the typed error. If accepted → INSERT INTO bids … RETURNING the new row, and a BidSnapshot is built from it. Drizzle ORM maps to the bids table defined in schema.ts.

DB · commit

11 COMMIT → PostgreSQL

The postgres-js pool (src/db/client.ts) commits the transaction. Only now is the bid durable and the lock released.

after commit only
Real-time

12 Broadcast the event src/server/auction-commands.ts → src/domain/events.ts

Records the bid-mutation duration metric, logs bid.accepted / bid.rejected, then calls publishAuctionEvent(event) on the in-process AuctionEventBus. Rejections are delivered only to the acting user. The typed result also returns up the stack (steps 5→1) to resolve the mutation.

↩ Real-time return path (back to every client)

SSE stream

13 Event bus → SSE handler src/routes/api.auctions.$auctionId.events.ts

Each connected client holds an open text/event-stream. Its subscribeToAuction() listener serialises the event as event: bid.accepted\ndata: {…} and enqueues it into the ReadableStream.

Client

14 Patch the cache src/routes/auctions.$auctionId.tsx

The EventSource listener for bid.accepted de-dupes by bidId, then queryClient.setQueryData prepends the bid and updates the highest-bid snapshot — the live bid chain updates instantly for all viewers. Aggregate dashboard/stats queries are invalidated rather than recomputed in the browser.
