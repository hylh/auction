Fish Auction POC implementation summary

This document summarizes the work completed in the session that started the auction proof of concept for the hylh/auction repository.

  • TanStack Start
  • TanStack Router
  • TanStack Query
  • PostgreSQL
  • Drizzle ORM
  • Zod
  • Server-Sent Events
  • Vitest
  • oxlint
  • oxfmt

Outcome

Implemented the first working vertical slice for a full-stack fish auction house POC. The application now has a TanStack Start scaffold, domain schema, seed data, transactional bid rules, real-time auction events, core UI routes, metrics, simulator plumbing, tests, and local setup documentation.

1. App scaffold

  • Created package.json with pnpm scripts.
  • Added TypeScript config, Vite config, and Vitest config.
  • Added TanStack Start root document, router setup, Query Client provider, navigation, and base CSS.
  • Added .gitignore for dependencies, build output, env files, and generated route tree.

2. Database and local infrastructure

  • Added Docker Compose PostgreSQL service with persistent volume and healthcheck.
  • Added Drizzle configuration, database client, schema, and initial SQL migration.
  • Modeled users, fish_items, auctions, bids, and sales.
  • Added indexes for status, species, timing, bid history, sales, and uniqueness for one sale per auction/winning bid.
  • Added deterministic seed data for demo sellers, buyers, admin, fish inventory, active/closed auctions, bids, and a completed sale.

3. Domain rules and services

  • Added shared constants for currency, curated species, demo users, demo fish, and demo auctions.
  • Added integer money and weight helpers: centsFromMajor and gramsFromKilograms.
  • Added Zod schemas for fish input, bid input, UUIDs, species, and auction close input.
  • Implemented typed bid rejection codes: STALE_BID, AUCTION_NOT_ACTIVE, SELLER_OWN_AUCTION, INSUFFICIENT_INCREMENT, and INVALID_AMOUNT.
  • Implemented transactional bid acceptance with auction row locking, current-highest-bid checks, seller permission checks, minimum increment validation, and post-commit event publication.
  • Implemented admin auction close behavior for sold and unsold outcomes.

4. Real-time events and observability

  • Added typed auction events for bid.accepted, bid.rejected, auction.closed, and sale.completed.
  • Added an in-process auction event broadcaster with subscribe and publish APIs.
  • Added per-auction SSE endpoint at /api/auctions/$auctionId/events.
  • Ensured rejected bid SSE events are only sent when the listener has the matching acting user id.
  • Added structured server logging helpers.
  • Added OpenTelemetry API span usage around bid mutations.
  • Added Prometheus-style metrics at /metrics.

5. User-facing routes

  • /: live operational dashboard showing active auctions, latest bids, recent sales, and inventory needing action.
  • /auctions/$auctionId: auction detail page with live bid chain, demo buyer selector, bid form, SSE subscription, Query cache patching for accepted bids, and aggregate invalidation.
  • /inventory/new: fish inventory form with curated species, kilograms, catch region, grade, seller, starting price, description, and optional image URL.
  • /admin: admin/history/statistics dashboard with total sales, average bid price, popular fish, auction close controls, completed sales, and bid history.

6. Simulator and documentation

  • Added simulator API route at /api/simulate that submits bids through the same service path as the app.
  • Added pnpm simulate script with configurable bid count and interval arguments.
  • Expanded README.md with local setup steps, routes, and script descriptions.
  • Added .env.example with default database URL and app origin.
