🐟 Fish Auction — Codebase Map & Glossary

Quick reference. Print-friendly. The canonical names used across all onboarding lessons.

Dependency rule: routes & components → server → domain & db. The domain layer imports nothing outward (no React, no DB).

Domain glossary

fish_items
Inventory. Weight stored as integer grams; price as integer cents.
auctions
Lifecycle & timing over a fish item. Statuses: scheduled, active, closed, unsold, withdrawn.
bids
Accepted bid history (immutable, ordered).
rejected_bids
Rejected attempts — audited, never silently dropped.
sales
Completed outcome = winning highest bid.
users
Seeded demo users; roles: seller, buyer, admin.
Server function
A typed RPC (createServerFn) imported by the client, run on the server.
Event bus
In-process pub/sub (domain/events.ts) keyed by auctionId.
SSE
Server-Sent Events: one-way server→browser stream over plain HTTP; auto-reconnects.
evaluateBid()
Pure accept/reject decision; no DB.
FOR UPDATE / row lock
Serializes concurrent bids on one auction so the highest bid isn't read stale.
setQueryData vs invalidateQueries
Patch the known detail change directly; invalidate aggregates to refetch.
Species (MVP)
salmon, cod, tuna, halibut, mackerel, trout, herring.
Inventory status
draft, listed, in_auction, sold, withdrawn.
Currency
NOK (single configured currency).
Demo users
sellerNorth, sellerFjord, buyerOslo, buyerBergen, admin (fixed UUIDs).

Bid error codes

Returned by evaluateBid(): stale bid, auction not active, seller bidding on own auction, insufficient increment, invalid amount. See src/domain/bid-rules.ts.
