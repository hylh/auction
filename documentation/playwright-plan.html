Playwright Implementation Plan

Status: In progress — Phase 1 setup underway (no CI in scope)
Created: 2026-06-05
Scope: End-to-end browser testing for the auction house application

Goals

  • Validate critical user journeys end-to-end in a real browser.
  • Cover flows that unit/integration tests cannot: real-time SSE updates in the browser, form interactions, navigation.
  • Enable reliable local E2E execution during development.
  • Keep tests fast and focused — no redundant coverage of business logic already covered by Vitest.

Installation

pnpm add -D @playwright/test
pnpm exec playwright install --with-deps chromium

Add playwright.config.ts at the project root. Use Chromium only initially to keep setup focused; add Firefox/WebKit later if needed.

Configuration

// playwright.config.ts
import { defineConfig } from '@playwright/test';

export default defineConfig({
  testDir: './e2e/tests',
  timeout: 30_000,
  retries: 0,
  use: {
    baseURL: 'http://localhost:3000',
    trace: 'on-first-retry',
    screenshot: 'only-on-failure',
  },
  webServer: {
    command: 'pnpm dev',
    url: 'http://localhost:3000',
    reuseExistingServer: true,
  },
  projects: [{ name: 'chromium' }],
});

Test Suites

1. Live Dashboard (dashboard.spec.ts)

  • Loads and displays active auctions.
  • Shows latest bids and recent sales.
  • Links to auction detail pages.

2. Auction Bidding (auction-bidding.spec.ts)

  • Navigate to an active auction detail page.
  • Place a valid bid — confirm it appears in the bid history.
  • Attempt a bid below the minimum increment — confirm rejection message.
  • Attempt a bid on own auction as seller — confirm rejection.
  • Auction closes — confirm UI reflects closed state.

3. Real-time Updates (realtime.spec.ts)

  • Open auction detail in two browser contexts (two users).
  • User A places a bid — confirm User B's page updates in real-time via SSE without reload.
  • Confirm bid amount and bidder name update correctly in both views.

4. Inventory (inventory.spec.ts)

  • Fill in the add fish inventory form with valid data — confirm item appears in inventory list.
  • Submit with missing required fields — confirm validation errors.
  • Submit with invalid weight (non-numeric) — confirm error.

5. Admin Dashboard (admin.spec.ts)

  • Admin user loads the admin dashboard.
  • Completed auctions and sales history are visible.
  • Statistics (total sales, average bid, popular species) render correctly.
  • Admin can close an active auction manually.

Database Fixtures

Before each test suite, reset and re-seed the database using a dedicated test seed. Use a Playwright fixture that calls the existing db:seed script or a lightweight programmatic seed via Drizzle.

// e2e/fixtures/db.ts
import { test as base } from '@playwright/test';

export const test = base.extend({
  seedDb: [async ({}, use) => {
    await execSync('pnpm db:seed:test', { stdio: 'inherit' });
    await use(undefined);
  }, { scope: 'worker', auto: true }],
});

CI Integration

Out of scope for this implementation phase. No GitHub Actions or other CI wiring will be added in this plan revision.

Implementation Phases

Phase 1 — Setup (current)
Install Playwright, add config, scaffold e2e/ directory, add pnpm scripts, update .gitignore for Playwright artifacts.
Phase 2 — Page Objects
Implement page object models for dashboard, auction detail, inventory form, and admin dashboard.
Phase 3 — Core Test Suites
Implement dashboard, bidding, inventory, and admin test suites. These can be developed in parallel.
Phase 4 — Real-time Tests
Implement multi-context SSE real-time update tests. These require two browser contexts and careful timing with expect.poll or event-driven assertions.
